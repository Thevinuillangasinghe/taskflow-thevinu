# TaskFlow Project Plan

## Week 1 — Requirements and Planning

- Create GitHub repository
- Create project documentation
- Write scope document
- Write user stories
- Set up Jira board
- Plan project milestones

Definition of Done:
Scope document, project plan, Jira board, and initial user stories completed.

---

## Week 2 — System Design

- Create ERD
- Create architecture diagram
- Design API contract
- Create wireframes
- Plan database schema

Definition of Done:
System architecture and database design fully documented.

---

## Week 3 — Backend Development

- Set up Fastify backend
- Configure Prisma ORM
- Implement authentication
- Implement workspace APIs
- Implement task APIs

Definition of Done:
Backend APIs functional and connected to database.

---

## Week 4 — Frontend Development

- Build login and signup pages
- Build workspace dashboard
- Build Kanban board UI
- Implement drag-and-drop functionality

Definition of Done:
Frontend connected to backend with working task management features.

---

## Week 5 — Testing and Deployment

- Write unit tests
- Write integration tests
- Write E2E tests
- Configure Docker
- Deploy frontend and backend

Definition of Done:
Application deployed publicly with passing tests.

---

## Week 6 — Maintenance and Handover

- Complete README
- Write runbook
- Create demo video
- Write retrospective
- Final presentation preparation

Definition of Done:
Project fully documented and ready for final review.
